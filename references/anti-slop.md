# Anti-Slop Rules

Use these rules to reduce generic AI-looking design.

## Anti-Patterns

### Generic AI SaaS Hero

- Vague headline about "transforming workflows."
- Abstract glowing object instead of product meaning.
- CTA that does not explain what happens next.
- No category, audience, or point of view.

### Random Gradients

- Purple or blue gradients with no connection to the product.
- Gradients used to create excitement because the brand has no idea.
- Backgrounds that reduce readability.

### Weak CTA

- Buttons labeled "Get started" with no context.
- Equal visual weight for every action.
- CTA placed before the user understands the value.

### Cluttered Card Grids

- Six feature cards that all say the same thing.
- Cards used as decoration instead of information structure.
- Icons that do not add meaning.

### Fake Glassmorphism

- Frosted panels that make text harder to read.
- Transparent layers used as a shortcut for "premium."
- Blurry depth with no information hierarchy.

### Boring Logos

- Generic initials in rounded squares.
- Sparkles, chat bubbles, abstract loops, or arrows with no brand idea.
- Marks that do not fit the product category.

### Inconsistent Color

- Accent colors used randomly.
- No roles for primary, surface, border, text, and semantic colors.
- Visual system changes from section to section.

### No Visual Hierarchy

- Every heading, card, and button has similar weight.
- Hero section does not answer what the product is.
- Important content appears after decorative content.

### Overused Icon Styles

- Sparkles for AI, lightning for speed, magic wand for automation, without a specific reason.
- Mixed icon weights or visual metaphors.
- Icons replacing clearer text.

### Meaningless Dashboard Mockups

- Random charts without labels, grouping, or believable use.
- Metrics that do not support a decision.
- Dashboards designed as background decoration rather than product proof.

## Prefer

- Category-specific brand signals.
- Clear product promise in the first screen.
- Intentional type scale and restrained color.
- Concrete UI examples instead of vague style words.
- Real interface structure over decorative filler.
- Specific do / don't rules an AI builder can follow.

## Audit Questions

- Could this belong to any AI startup?
- Does the logo communicate category, behavior, or attitude?
- Does the palette have roles, or only vibes?
- Is the hero obvious in five seconds?
- Are the cards saying different things, or repeating the same pattern?
- Would a builder know exactly what to implement next?

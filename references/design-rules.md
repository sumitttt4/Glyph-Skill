# Design Rules

Use these rules when creating `/glyph design`, `/glyph audit`, or `/glyph kit` outputs.

## First Screen

- Make the product category obvious immediately.
- Give the hero one clear visual idea.
- Place the primary CTA where intent is strongest.
- Avoid hero sections that look like interchangeable SaaS templates.

## Layout

- Create rhythm through contrast in section density.
- Use whitespace to clarify relationships.
- Keep repeated cards consistent but not lifeless.
- Align component style with the brand direction.

## Components

- Buttons should have a clear action hierarchy.
- Cards should frame meaningful choices or content.
- Dashboards should optimize scanning and comparison.
- Empty states, labels, and microcopy should preserve the brand voice.

## Visual Hierarchy

- Decide what the user should notice first, second, and third.
- Use scale, weight, spacing, and contrast intentionally.
- Avoid making every section equally loud.
- Make every visual choice explainable.

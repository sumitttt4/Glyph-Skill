# Design Rules

Use these rules when creating `/glyph design`, `/glyph audit`, or `/glyph kit` outputs.

## Landing Pages

- Make the product category obvious immediately.
- Give the hero one clear visual idea.
- Place the primary CTA where intent is strongest.
- Avoid hero sections that look like interchangeable SaaS templates.
- Use section rhythm: promise, proof, mechanism, examples, use cases, CTA.
- Show the product or a product-relevant visual system early.
- Do not bury the most important differentiator below generic benefit cards.

## SaaS UI

- Prioritize scanning, repeated use, and clear state over decorative composition.
- Use compact density for operational tools and more whitespace for strategic or creative tools.
- Keep navigation predictable and labels literal.
- Make primary actions visually consistent across screens.

## Dashboards

- Design dashboards around decisions, not charts.
- Group metrics by question or workflow.
- Make current state, trend, and next action easy to compare.
- Use table density, filters, and empty states intentionally.
- Avoid meaningless dashboard mockups filled with random graphs.

## Hero Sections

- The headline should make the product category or offer obvious.
- Supporting copy should explain who it is for and why it matters.
- The hero visual should reinforce the product's core behavior.
- Keep CTA hierarchy clear: one primary action, one secondary action when needed.

## Layout

- Create rhythm through contrast in section density.
- Use whitespace to clarify relationships.
- Keep repeated cards consistent but not lifeless.
- Align component style with the brand direction.
- Use max-widths and grid structure to make content feel intentionally placed.
- Avoid floating card piles when a simple section would communicate better.

## Components

- Buttons should have a clear action hierarchy.
- Cards should frame meaningful choices or content.
- Dashboards should optimize scanning and comparison.
- Empty states, labels, and microcopy should preserve the brand voice.
- Cards need a reason to exist: comparison, grouping, preview, selection, or emphasis.
- Buttons should use color for importance, not decoration.
- Icon style should match the brand tone and remain consistent.

## Visual Hierarchy

- Decide what the user should notice first, second, and third.
- Use scale, weight, spacing, and contrast intentionally.
- Avoid making every section equally loud.
- Make every visual choice explainable.

## Spacing

- Use spacing to show relationships between elements.
- Increase section spacing when the story changes.
- Keep component interior padding consistent across a UI family.
- Avoid cramped dashboards and oversized marketing blocks that waste useful space.

## Responsive Behavior

- Preserve the primary message and CTA on mobile.
- Collapse grids into readable single-column flows.
- Keep code blocks, commands, and long labels wrapping cleanly.
- Avoid visual mockups that become unreadable or dominate the screen on mobile.
- Ensure tap targets are comfortable and nav remains simple.

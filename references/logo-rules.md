# Logo Rules

A logo should make the product recognizable, fit its category, and work at small sizes. Most AI-generated logos fail because they pick a symbol first and then try to justify it. Start with the product logic.

---

## Logo Decision Process

Before choosing a logo direction, answer:

1. What category is the product? The logo must not confuse the category.
2. Is the product name distinctive enough to carry a wordmark?
3. Does the product need an icon (app stores, social avatars, browser tabs)?
4. What stage is the product? Early stage usually benefits from wordmarks over symbols.
5. What is the personality: technical, warm, editorial, playful, premium, or institutional?
6. What does the competition use, and where is the visual space to differentiate?

---

## Logo Types

### Wordmark
A typeset version of the product name with no symbol.

- Use when: the product name is short, distinctive, and needs clarity before memorability
- Use when: the brand is early stage and recognition matters more than visual shorthand
- Use when: the product is primarily text or document-based (note tools, writing tools, docs)
- Avoid when: the name is too long (5+ words) or hard to set typographically
- Style rules: choose a typeface that reflects the product personality. Fintech = geometric sans. Creator = humanist sans. Editorial = refined serif.

### Symbol + Wordmark (Combination Mark)
A recognizable icon paired with the product name.

- Use when: the product has a strong app presence (mobile, desktop, extension)
- Use when: the brand will need to separate the icon for social media avatars
- Use when: the symbol can encode a clear metaphor or behavior
- Rule: the symbol and wordmark must feel like they belong to the same visual system

### Symbol / Logomark (Icon Only)
A standalone icon that represents the brand.

- Use when: brand recognition is already established
- Use when: the product behavior can be reduced to a simple, ownable shape
- Avoid at early stage unless the symbol idea is unusually strong
- Rule: must work at 16×16px without losing legibility

### Monogram
A typographic treatment of one or two initials.

- Use when: the initials form a visually strong shape
- Use when: the brand needs formality or institutional weight
- Avoid when: the initials are common combinations (AI, SB, TM)
- Avoid when: the monogram becomes visually complex or confusing

## Symbol Direction Rules

When a symbol is appropriate, guide the idea logically:

- **Metaphor-driven**: the symbol encodes a concept directly tied to what the product does
  - Example: a slash mark for a code or command tool, a ruled line for a document tool
- **Behavior-driven**: the symbol captures how the product moves or transforms data
  - Example: a loop for iteration, an arrow for progression, a grid for organization
- **Category-driven**: the symbol uses the visual language of the industry
  - Example: a lock for security, a chart line for analytics, a pen for writing

### Construction Logic
Always explain the construction:

- Geometry: what shapes are used and why
- Proportion: how the symbol relates to the wordmark in size
- Stroke: consistent stroke weight or filled form
- Grid: does the symbol sit on a clear base grid
- Scalability: does it simplify cleanly at small sizes

---

## Output Format

When generating a logo direction, output in this format:

```
Logo type:          [Wordmark / Symbol + Wordmark / Monogram / Symbol only]
Symbol idea:        [What the mark represents and why it fits]
Wordmark style:     [Typeface direction, weight, any modifications]
Shape language:     [Geometric / Organic / Humanist / Technical]
Construction:       [Geometry, proportion, stroke weight, grid logic]
App icon direction: [How it adapts for square icon format]
Color application:  [How brand colors apply to the logo]
Avoid:              [Specific things to not do with this logo]
```

## Category Signal Guide

| Category | Logo Direction |
|---|---|
| Fintech / Banking | Geometric sans wordmark, minimal symbol, no playful marks |
| Developer Tool | Monospace-influenced wordmark, slash, terminal, or syntax metaphor |
| AI SaaS | Clean wordmark or minimal mark — avoid sparkles and brain icons |
| Creator Tool | Humanist sans, warmer mark, more expressive shape language |
| Healthcare | Rounded, accessible mark, no sharp geometric aggression |
| Luxury | Refined serif or high-contrast sans wordmark, no playful symbol |
| Analytics | Geometric mark, data-adjacent metaphor (line, bar, node) |
| Education | Friendly rounded sans, bright accent, accessible mark |
| Enterprise | Strong wordmark, institutional feel, no decorative symbol |

---

## App Icon Rules

When the logo needs an app icon version:

- Use the symbol or first letter, not the full wordmark
- Must be legible at 32×32px and 16×16px
- Use a single background color from the brand palette
- Avoid complex detail that disappears at small sizes
- Test at small size before finalizing

---

## Never Do

- Do not use generic sparkles to signal AI.
- Do not use a brain icon for AI or intelligence products.
- Do not use a shield for every security product.
- Do not use a hexagon as a default technical mark.
- Do not use abstract blobs with no construction logic.
- Do not use a chat bubble for every communication tool.
- Do not use a lightning bolt for speed unless the brand rationale is very strong.
- Do not choose initials in a rounded square and call it a logo.
- Do not mix symbol styles (outlined + filled in the same mark).
- Do not make a logo that does not survive being reproduced at 24×24px.
- Do not explain the logo as "modern and minimal" without describing what it actually is.
